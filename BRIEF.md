# The Pearl 项目简报

> 所有开发、修改和评审都必须与本文件对齐。每次新对话应先阅读本文件和 `CLAUDE.md`。

## 1. 项目目标

The Pearl 是一个面向家电电商客服的自进化 Agent 黑客松 Demo。

当前唯一需要证明的核心价值是：

> 一次不够好的客服回答，经过人工纠正、经验提炼和批准后，可以在下一次相似咨询中复用，让回答产生肉眼可见、可解释、可验收的改善。

本项目不是完整的生产级客服系统。

## 2. 优先级

1. Demo 稳定可运行。
2. 进化前后差异清楚。
3. 5 分钟内能向评委讲明白。
4. 有备用截图或录屏，现场失效时仍能完成展示。

## 3. 技术栈与入口

- **语言**：Python 3.12+
- **框架**：Streamlit
- **数据**：本地 JSON 文件
- **集成**：GEP (GenAI Evolution Platform) / EvoMap
- **Node 依赖**：EvoMap Evolver、GEP SDK、Ajv
- **Demo 入口**：`app.py`
- **启动命令**：`.venv/bin/streamlit run app.py`
- **测试命令**：`.venv/bin/python -m unittest discover -s tests -v`

## 4. P0 范围（必须保住）

- 客户聊天页面
- 人工审核与纠正
- 候选 Capsule 的生成和批准
- 相似问题复用
- 进化前后对比
- 简明的进化事件或看板
- 一键重置或可重复演示能力
- 固定测试、路演脚本和备用演示材料

## 5. 本次默认不做

- 真实电商平台接入
- 生产级登录、权限、工单和数据库
- 无人审核的自动发布
- 大规模模型训练
- 完整知识图谱
- 第三个业务场景

## 6. 固定 Demo 剧本

**首次问题**：洗衣机脱水时一直乱跳，是不是坏了？

**初始回答的缺陷**：没有明确排查顺序；没有覆盖运输螺栓、地面和脚垫、衣物偏载；没有清楚说明停止使用和转人工的边界。

**人工纠正后的经验**：
1. 新机先检查运输螺栓是否拆除
2. 检查地面是否坚硬平整、四个脚垫是否稳定
3. 检查衣物是否缠绕或偏在一侧
4. 空桶仍剧烈跳动，或伴随异响、漏水、焦味时停止使用并转人工
5. 不擅自承诺故障结论、赔付、维修或上门结果

**相似问题**：洗衣机一脱水就晃得厉害，还会移位，怎么办？

**验收重点**：新版回答必须比初始回答更具体、更安全，并能解释「为什么发生了变化」。

## 7. 最低验收标准

1. 测试全部通过
2. 固定主链路连续完整运行 3 次
3. 进化前后差异无需解释代码也能看懂
4. 状态重置后可再次演示
5. 没有把模拟能力包装成真实能力

## 8. 数据与安全边界

- 不向 EvoMap、公共仓库或外部服务上传原始客户对话、姓名、电话、地址等业务秘密
- 实验优先使用固定、脱敏、最小化样本
- 推送 GitHub 前必须单独确认仓库可见性和上传范围

## 9. 伙伴上手步骤

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
npm install
cp .env.example .env
.venv/bin/streamlit run app.py
```
